# Локальный поисковый движок на Java Spring Boot

## Описание

Этот проект - это локальный поисковый движок, который может обходить и индексировать страницы заданных сайтов и находить наиболее релевантные страницы по любому поисковому запросу. Этот проект решает проблему поиска информации на ограниченном наборе сайтов.

## Стэк технологий

Этот проект использует следующие технологии:

- Spring Boot - фреймворк для создания веб-приложений на Java
- Maven - инструмент для управления зависимостями и сборки проекта
- Thymeleaf - шаблонизатор для генерации HTML-страниц
- Lombok - библиотека для упрощения написания кода на Java
- MySQL - реляционная база данных для хранения информации о сайтах и страницах
- JSOUP - библиотека для парсинга HTML-документов
- jpa - спецификация для работы с базами данных на Java
- LuceneMorphology - библиотека для лемматизации слов

## Инструкция по локальному запуску проекта

Для запуска проекта необходимо выполнить следующие шаги:

1. Установить и запустить MySQL-сервер на локальном компьютере.
2. Создать новую базу данных с именем `search_engine` и кодировкой `utf8mb4/utf8mb4_general_ci`.
3. Скопировать файл `application.yaml` в папку с проектом и отредактировать его, указав свои параметры подключения к базе данных (имя пользователя, пароль и порт).
4. Скопировать файл `SearchEngine-1.0-SNAPSHOT.jar` в папку с проектом.
5. Открыть командную строку и перейти в папку с проектом.
6. Ввести команду `java -jar SearchEngine-1.0-SNAPSHOT.jar` и нажать Enter.
7. Открыть браузер и перейти по адресу `http://localhost:8080`.

## Принципы работы поискового движка

Поисковый движок работает по следующему алгоритму:

1. В конфигурационном файле `application.yaml` задаются адреса сайтов, по которым движок осуществляет поиск.
2. Поисковый движок самостоятельно обходит все страницы заданных сайтов и индексирует их (создавая так называемый индекс), так чтобы потом он мог находить наиболее релевантные страницы по любому поисковому запросу. Для этого он использует библиотеку JSOUP для парсинга HTML-документов и библиотеку LuceneMorphology для лемматизации слов на русском языке. Информация об индексированных страницах сохраняется в базе данных MySQL.
3. Пользователь присылает запрос через API движка. Запрос — это набор слов, по которым нужно найти страницы сайта. Запрос определённым образом трансформируется в список слов, переведённых в базовую форму. Например, для существительных — именительный падеж, единственное число.
4. В индексе ищутся страницы, на которых встречаются все эти слова. Результаты поиска ранжируются, сортируются и отдаются пользователю в виде HTML-страницы, сгенерированной с помощью шаблонизатора Thymeleaf.
